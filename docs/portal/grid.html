Every agent decision passes the confidence gate

Grid intercepts traffic, routes it through agent pipelines, and enforces one non-negotiable rule: an agent that isn't confident enough cannot auto-enforce. Below the floor, its decision is turned into a DEFER and routed to a human. The agent proposes; the runtime disposes.

Runtime note — Grid runs on HALO. The gate, tool registry, and audit you see here aren't re-implemented inside each Grid agent. Grid's agent mesh executes inside HALO — the vendor-neutral agent runtime → which owns the confidence gate, the default-deny tool registry, and safe-failure defaults. HALO is a separate peer platform Grid consumes; it is not part of Aether. This portal shows the behaviour Grid inherits from it.

The gate logic, exactly

  1. SUGGEST-authority agents never auto-enforce. Whatever the confidence, a SUGGEST decision always routes to a human. "AI drafts, humans approve" is a property of the authority ladder, not app code.
  2. Enforcement needs confidence ≥ the floor (0.80). Below it, autoEnforced=false and the action becomes DEFER.
  3. BLOCK is the highest authority — it needs ≥ 0.95. Stopping traffic is the most consequential act, so an under-confident BLOCK defers to a human rather than firing.
  4. A DEFER routes into Aether Flow's approval queue. That's the seam between the two platforms — Grid decides it can't decide, Flow gets a human to.
  5. Once the outcome is known, a feedback signal flows to Core. CORRECT / INCORRECT reinforces or weakens the memories that informed the decision.
