Retrieval that never leaves the collection

Vault turns documents into knowledge. A RAG query is embedded, vector-searched against document chunks inside one collection, and the top matches are assembled into a length-bounded context for an agent's prompt. Vault retrieves; the caller generates. The collection boundary is never crossed.

What the pipeline guarantees

  1. Retrieval is scoped to one collection. The query only ever sees chunks whose collection_id matches. Switch collections and the previous documents don't exist to this query — no cross-collection, no cross-tenant path.
  2. top-K is clamped. Ask for more than MAX_TOP_K and the pipeline caps it — an agent can't drain the store in one call.
  3. Context is length-bounded. Chunks are added in rank order until the character budget is reached; the rest are dropped. The prompt an agent receives is always bounded.
  4. Vault retrieves; it does not generate. The assembled context is handed back for the caller's LLM. Generation happens in the agent, governed by Grid.
Bounded context: Vault owns documents and the knowledge derived from them. It holds no personal memory (Core) and no shared team memory (Memory).
