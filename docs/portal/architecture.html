Five platforms, one cognitive fabric

Each platform owns exactly one capability and never reaches into another's store. They compose through clean seams — REST contracts, the DEFER handoff, and the feedback loop. Here's how a single request flows through all of them.

Dependency direction

Dependencies point downward. A platform may consume the one below it through a port or REST contract; it never reaches up or sideways into another's internals.

The seams that make it compose

🔗 REST context contract
Grid asks Core for a user's assembled personalContext over REST. Core stays the only owner of personal memory; Grid never keeps its own copy.
⚡ The DEFER seam
When Grid's confidence gate can't auto-enforce, the decision becomes a bounded DeferredDecision and lands in Flow's approval queue — no Grid internals cross.
🔄 The feedback loop
Once a decision's outcome is known, Grid publishes CORRECT / INCORRECT to Core, reinforcing or weakening the memories that informed it. The system learns.
🌐 Federation projection
Memory shares only FEDERATED memories across instances, projected to bounded summaries — identity never crosses the boundary.

Built by a method, run on a runtime

AIEL — the methodology

Every Aether component is engineered and governed by the Aether Intelligence Engineering Lifecycle — a 12-phase lifecycle (Strategy → … → Retirement). It's doc-only: it specifies the agent contract, the authority ladder, the confidence gate, and the governance controls you saw enforced in the simulations. It builds no runtime itself.

HALO — the runtime peer platform

Grid's agent mesh executes inside HALO — a vendor-neutral agent runtime — which owns the confidence gate, the default-deny tool registry, audit, and safe-failure defaults that AIEL specifies. HALO is not part of Aether: it's an independent peer that Grid (and other products) consume so they don't re-implement agent execution. This portal shows the behaviour Grid inherits from it.

Why the boundaries are strict: one capability per platform, consumed through ports or REST, never re-implemented. That's what lets any platform run standalone, be replaced, or scale independently — and it's why Aether, HALO, and the SDLC tooling stay separate products rather than one tangled codebase.
