Human approval, with a clock on it

Flow drives multi-step human-AI processes through a persisted state machine. When it reaches a HUMAN_APPROVAL step it raises an approval task with an SLA deadline. Miss the deadline and an escalation sweep flags it — it marks visibility, it never auto-decides. And when Grid DEFERs a decision, it lands right here.

The rules the engine enforces

  1. Every transition is persisted. The instance's state survives a restart — that's the State Persistence capability. Here each step change is logged as it commits.
  2. The human gate raises a task with an SLA. Reaching HUMAN_APPROVAL creates a PENDING task with a deadline; the instance waits on a human, not a timer.
  3. Escalation marks; it never decides. When the SLA breaches, the sweep flips PENDING → ESCALATED to raise visibility. It never approves, rejects, or deletes — a human always decides.
  4. Grid DEFERs arrive as a bounded projection. A deferred decision enters the queue carrying only a summary and coarse provenance — no Grid internals, no raw payload, no PII.
Bounded context: Flow owns orchestration and human approval. It stores no personal memory (Core), shared memory (Memory), or documents (Vault). It integrates with Grid only at the DEFER seam.
