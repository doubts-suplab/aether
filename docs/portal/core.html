Aether Core · The Brain · Port 8082

Personal memory that reinforces and decays

Core is the personal cognitive engine. It stores four kinds of memory — episodic, semantic, procedural, emotional — each with a strength. Recall a memory and it strengthens; leave it idle and it decays. What stays strong is what gets assembled into the personal context an agent sees.

SimulationMemory reinforcement & decay
Recall a memory to reinforce it (strength += reinforcement, capped at 1.0). Advance time to decay every idle memory. Below the archive threshold a memory dims — the system naturally forgets what stopped mattering.

What just happened

  1. Recall reinforces. Every access nudges strength up and refreshes the memory's recency — memories that matter stay relevant. This is the same signal Grid feeds back after a decision outcome is known.
  2. Idle memories decay. The scheduled decay sweep lowers the strength of anything not recalled. Nothing is deleted; it simply fades below the archive threshold.
  3. Context is assembled, not dumped. The snapshot an agent receives is the top memories by current strength plus the live session — a synthesized view, scoped to one user, never the whole store.
Bounded context: Core owns personal memory and nothing else. Shared team memory lives in Aether Memory; documents and knowledge live in Aether Vault. Every query here is scoped to a single userId — there is no cross-user read path.
